# AI 協作脈絡檔 - 像素塔防遊戲

## 專案目標
製作一款瀏覽器即可遊玩的像素風塔防遊戲。開發者不懂程式與美術，所有邏輯由 AI 輔助生成。
核心訴求：遊戲架構必須**健康、可擴展、不硬編碼**。

## 開發環境
- 編輯器：VSCode
- 測試方式：Live Server 外掛
- 語言：純 HTML/CSS/JavaScript（無框架）
- 資料庫：Firebase Realtime Database（存檔、排行榜、好友）

## 檔案結構與職責速查表

### 📁 根目錄
| 檔案 | 內容摘要 |
|------|----------|
| `index.html` | 頁面入口，含主畫面(lobby)、遊戲畫面(game)、結算畫面(result)。所有 JS 載入順序在此定義。 |
| `css/style.css` | 全站樣式，含手機橫豎屏自適應、主畫面摺疊面板樣式。 |

### 📁 js/（核心腳本）
| 檔案 | 職責 | 關鍵內容 |
|------|------|----------|
| `config.js` | 🔥 **唯一設定檔** | 所有遊戲數值：地圖、塔、敵人、波次、配方（含唯一 `id`）、經濟、升級、抽卡機率。 |
| `utils.js` | 工具函式 | 距離計算、大/小塔位查詢、合成輔助、大格子交換(`swapBigSlots`)。 |
| `enemy.js` | 敵人邏輯 | 生成(`spawnEnemy`)、動態血量、移動、繪製、緩速/閃白。 |
| `effects.js` | 特效管理 | 波紋(`spawnWaveEffect`)、浮動文字(`spawnFloatingText`)、更新與繪製。 |
| `projectile-effects.js` | 投射物命中效果 | `PROJECTILE_EFFECTS` 字典，每個效果獨立函式（濺射、緩速、燃燒、凍結等），支援複合效果。 |
| `projectile.js` | 投射物邏輯 | 子彈生成、移動、碰撞、濺射、緩速、擊殺獎勵。 |
| `firebase.js` | Firebase 封裝 | 存檔代碼、排行榜、好友雲端同步。 |
| `storage.js` | 局外永久資料 | `PlayerData`管理、升級、好友、本地/雲端同步。 |

### 📁 js/tower/（塔模組）
| 檔案 | 職責 | 關鍵函式 |
|------|------|----------|
| `tower-core.js` | 塔核心 | `placeTower`(建造)、`sellTower`(販賣)、`drawTowers`(繪製)、`getTowerAt`(點擊偵測)、`getTowerCenter`(中心座標)。 |
| `tower-actions.js` | 塔行動 | `applyTowerEffects`(光環效果)、`updateTowers`(攻擊索敵)。 |

### 📁 js/game/（遊戲主程式拆分）
| 檔案 | 職責 | 關鍵內容 |
|------|------|----------|
| `game-state.js` | 狀態宣告 | Canvas、ctx、所有遊戲變數（`gold`, `towers`, `enemies`, `wave`, 拖曳狀態等）。 |
| `game-synth.js` | 合成進化 | `checkEvolution`、`evolveBigSlot`、`checkAndEvolveAt`。 |
| `game-update.js` | 主迴圈 | `update()`、`nextWave()`、`updateUI()`、`gameLoop()`。 |
| `game-draw.js` | 繪圖 | `draw()`（地圖、塔、敵人、UI面板）、`drawBuildPanel()`。 |
| `game-events.js` | 事件處理 | 座標轉換、`handleClick`、所有 pointer 事件（拖曳、建造、販賣）。 |
| `game-lobby.js` | 局外大廳 + 頁面導航 | `switchPage()` 切換五個頁面（成長/圖鑑/首頁/好友/排行）、`renderLobby()`、`renderCollection()`（含稀有度分類篩選）、設定彈窗（存檔代碼/複製/還原）、`endGame()`。 |

## 載入順序（重要！）
1. `config.js` → 2. `utils.js` → 3. `firebase.js` → 4. `storage.js` → 5. `game-state.js`
6. `tower-core.js` → 7. `tower-actions.js` → 8. `enemy.js` → 9. `effects.js` → 9. `projectile-effects.js` → 10. `projectile.js`
11. `game-synth.js` → 12. `game-draw.js` → 13. `game-update.js` → 14. `game-events.js` → 15. `game-lobby.js`

## 健康架構原則（AI 必須遵守）
1. **所有遊戲數值、塔種類、敵人屬性、波次設定、配方、永久升級數值，一律定義在 `js/config.js` 的 `CONFIG` 物件中**，其他檔案只讀不寫。
2. 程式碼不可出現魔術數字（hardcode），調整平衡只需改設定檔。
3. 新增塔/敵人/配方/升級時，僅擴充 `CONFIG` 中的陣列或物件，並在對應邏輯模組中讀取。
4. 所有修改必須明確指出「改哪個檔案、哪個區塊」，保持模組獨立。
5. 與人類溝通時，絕對不使用程式術語，用生活化語言說明。
6. 合成檢查僅在塔建造或拖曳結束後觸發，禁止每幀全場掃描。
7. 所有 Canvas 座標需透過 `getCanvasCoords` 轉換，以支援手機自適應。
8. **配方、升級、存檔之間的索引必須使用固定 `id`，不可使用可變的名稱（`name`）**，否則改名會導致玩家進度遺失。
9. **新增塔的命中效果（濺射、緩速、燃燒等）必須定義在 `projectile-effects.js` 的 `PROJECTILE_EFFECTS` 字典中**，`updateProjectiles` 僅負責遍歷 `effects` 陣列並查表呼叫，禁止在 `projectile.js` 中寫 `if-else` 分支處理具體效果。

## 目前遊戲完整功能清單

### 主畫面與局外系統（全螢幕 App 風格）

#### 整體布局
- 全螢幕深色主題，底部固定導航列（5 個頁籤），右上角齒輪設定按鈕。
- 導航順序（由左至右）：⬆️ 成長 → 📚 圖鑑 → 🏠 首頁 → 👥 好友 → 🏆 排行。
- 頂部固定顯示：星塵、最高波次、精華。

#### 首頁（中間）
- 顯示遊戲標題、星塵/最高波次/精華三項數據。
- 大型「開始遊戲」按鈕。
- 顯示目前已解鎖配方數量（例如：8/15）。

#### 成長頁面（最左）
- 上方：「抽取配方」按鈕，消耗 1 精華隨機獲得配方，重複轉碎片。
- 下方：永久升級列表（起始金幣、塔傷害加成、稀有機率加成），以卡片形式呈現。

#### 圖鑑頁面（左二）
- 稀有度分類標籤列：全部 / 普通 / 稀有 / 史詩 / 傳說 / 神話。
- 點擊標籤即時過濾顯示對應階層的配方。
- 每個配方顯示：持有狀態（✅/🔒）、名稱、階層、等級、碎片數量。
- 若碎片足夠升級，顯示「升級」按鈕。

#### 好友頁面（右二）
- 輸入好友代碼 + 新增按鈕。
- 好友列表顯示代碼與最高波次，右側有刪除按鈕。

#### 排行頁面（最右）
- 顯示 Firebase 排行榜前 10 名。
- 自己的紀錄高亮顯示。

#### 設定彈窗（點擊右上角齒輪）
- 顯示存檔代碼（大號字體，方便複製）。
- 「複製代碼」按鈕。
- 還原存檔區：輸入代碼 + 還原按鈕。
- 預留未來擴充（音效、語言等）。

### 地圖與建造
- 480×320 環形路徑地圖，中央 5×3 大塔位（每大塔位 2×2 小格，共 60 個建造點）。
- 大塔位粗邊框，小塔位虛線標示，建造時黃色高亮選中格。
- 點擊空小塔位 → 三選一類別面板（攻擊🔴/干擾🔵/輔助💎）。
- 動態建造費用（基礎 50，每次 +15，上限 350），販賣固定返還 30 金幣。

### 素材塔系統
- 三色素材塔，各有攻擊/緩速/賺金能力。
- 素材品質：一般(60%)、標準(30%)、完美(10%)，影響外觀與數值。
- 拖曳移動小塔（可與小塔/大塔交換），大塔可與整格互換。

### 合成進化（配方塔）
- 大格子內四小塔滿足精確配方（位置、顏色、品質）→ 自動合成大塔（不可販賣、可移動）。
- 配方分五階層：普通/稀有/史詩/傳說/神話，由抽取解鎖。
- 進化大塔支援複合效果（`effects` 陣列），例如 `['splash', 'slow']` 可同時觸發濺射與緩速。
- 配方可花費碎片升級（1~5級），提升傷害/範圍/攻速/金幣產量。

### 敵人與波次
- 敵人種類：普通、高速、重型、Boss（每10波）。
- 動態血量（隨波次分段成長），沿環形路徑循環，數量達上限(50)失敗。
- 波次固定 15 秒自動推進，每 5 波獎勵金幣，Boss 擊殺給鑽石+精華。

### 局外成長
- **星塵**：遊戲結束結算，用於永久升級（起始金幣、傷害加成、稀有機率）。
- **精華**：擊殺 Boss 獲得，用於抽取配方塔。
- **鑽石**：目前累積，未來可設計用途。

### 抽卡與圖鑑
- 花費 1 精華抽取配方（依階層權重隨機），重複轉碎片。
- 圖鑑以卡片形式顯示所有配方：預覽色塊、名稱、階層、等級、血條式進度條（碎片/需求）。
- 點擊卡片彈出獨立視窗（Modal），顯示：
  - 大圖示、名稱、階層標籤
  - 客製化描述文字
  - 合成配方（2×2 座標位置，標示左上/右上/左下/右下）
  - 當前等級的實際數值（傷害/範圍/攻速/金幣）
  - 可升級時顯示綠色「+增加量」與「升級」按鈕
  - 各級升級效果列表（自 Lv.1 至 Lv.5，當前等級高亮）
- 配方稀有度分類標籤：全部 / 普通 / 稀有 / 史詩 / 傳說 / 神話。

### 雲端與社交
- Firebase 存檔代碼（跨裝置還原）、排行榜（最高波次前10名）、好友系統（透過代碼新增/刪除）。

### 手機支援
- Pointer Events 觸控，Canvas 自適應縮放，橫屏優化，直屏提示旋轉。

### 特效
- 波紋（建造/合成/光環/交換）、浮動文字（賺金）、命中閃白、拖曳半透明預覽。

### 🧮 星塵結算公式
星塵 = 波次 × 10 + Boss擊殺數 × 50 + ⌊總擊殺數 / 100⌋ × 5

## 🐛 常見錯誤與排查指引（供 AI 對照）

| 錯誤訊息 | 可能原因 | 檢查方式 |
|----------|----------|----------|
| `xxx is not defined` | 函式定義的檔案尚未載入 | 檢查 `index.html` 載入順序，確認該檔案在呼叫者之前 |
| `Cannot read property 'x' of null` | `getBigSlotIndex` 回傳 null | 確認點擊座標確實在有效的大格子內 |
| `CONFIG is not defined` | `config.js` 未載入或載入順序錯誤 | 確認 `config.js` 是第一個載入的 script |
| 波次不會推進 | `waveTimer` 可能被設為 0 或負值 | 檢查 `nextWave()` 是否被正確呼叫 |
| 塔不會攻擊 | `fireRate` 或 `cooldown` 設定異常 | 檢查 `updateTowers()` 中的 `cooldown` 遞減邏輯 |
| 合成未觸發 | 配方未解鎖或品質不符 | 確認 `PlayerData.unlockedRecipes` 中是否有該配方 |
| 手機點擊偏移 | Canvas 縮放未轉換座標 | 確認所有事件都使用 `getCanvasCoords` |
| 新塔的數值異常（如金幣、傷害暴增） | `RECIPE_UPGRADE_EFFECTS` 中誤用絕對數值而非倍率 | 確認所有升級陣列為倍率，與 `evolveBigSlot` 中的乘法公式一致 |

## 📦 模組相依關係（由上而下 = 由底層到應用層）

config.js ← 唯一資料來源，所有人依賴它
↓
utils.js ← 純工具，無依賴
↓
firebase.js / storage.js ← 依賴 config，處理資料持久化
↓
game-state.js ← 宣告所有遊戲變數（gold, towers, enemies）
↓
tower-core.js ← 依賴 state（讀寫 towers, gold）
↓
tower-actions.js ← 依賴 tower-core + state（讀取 towers, enemies）
↓
enemy.js / effects.js / projectile.js ← 依賴 state（讀寫 enemies）
↓
game-synth.js / game-draw.js / game-update.js / game-events.js / game-lobby.js


## 🧩 檔案分類（修改時的安全邊界）

| 分類 | 檔案 | 修改風險 |
|------|------|----------|
| 🔥 **純資料** | `config.js` | 安全，只改數值不影響邏輯 |
| 🛠️ **工具** | `utils.js` | 安全，函式獨立 |
| 🧠 **邏輯** | `enemy.js`, `tower-actions.js`, `game-synth.js`, `game-update.js` | 修改時需注意與 state 的互動 |
| 🎨 **繪圖** | `game-draw.js`, `effects.js` | 修改時不影響遊戲邏輯，但可能影響視覺回饋 |
| 🖱️ **互動** | `game-events.js` | 修改時需確保與 `handleClick` 和拖曳邏輯一致 |
| 🏠 **UI / 大廳** | `game-lobby.js` | 與遊戲核心邏輯完全隔離，修改最安全 |
